# AI Workflow 项目协作规则

- 默认使用中文编写面向人的说明、报告、提交信息和文档。
- CLI 默认只生成可审阅产物，不直接修改目标仓库、不自动应用 patch、不提交代码。
- 校验命令必须从 `config/workflow.yaml` 读取，并且只能在用户指定的目标仓库目录内执行。
- 禁止在工作流中执行明显破坏性命令，例如 `git reset --hard`、`git clean`、`rm -rf`、`del /f`、`format`。
- 新增节点或模型适配时，必须保证 mock 模式仍可离线跑通测试。
- 所有工作流运行产物写入 `runs/<run_id>/`，不要混入目标仓库源码目录。
